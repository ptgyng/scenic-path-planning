# 景区路径规划系统 · Scenic Path Planning

基于图论算法的景区路径规划系统，支持导游线路生成(DFS)、回路检测、最短路径查询(Floyd)。

## 功能一览

| 编号 | 功能 | 核心算法 |
|------|------|---------|
| 1 | 创建景区景点图 | 邻接表存储 |
| 2 | 展示景区景点分布图 | 邻接矩阵输出 |
| 3 | 导游路线 | 深度优先遍历(DFS) |
| 4 | 导游路线中的回路 | 拓扑排序检测 |
| 5 | 两个景点间的最短路径和距离 | Floyd 动态规划 |

## 算法简介

### DFS 遍历

- **思想**: 深度优先，先访问一个邻居，递归深入直到无法继续，再回溯
- **应用**: 生成导游线路，覆盖所有可达景点

### 回路检测（拓扑排序）

- **思想**: 计算每个节点的入度，入度为 0 的节点入栈，依次删除其出边
- **判断**: 如果最终处理的节点数小于总节点数，说明存在回路

### Floyd 算法

- **思想**: 动态规划，`dist[k][i][j]` 表示 i→j 且中间节点仅允许使用 {0..k} 的最短距离
- **递推式**: `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`
- **复杂度**: O(n³)

## 设计要点

1. **邻接表存储**: 用单链表组织每个顶点的邻接边，稀疏图下空间效率优于邻接矩阵
2. **头文件保护**: `#ifndef GLOBAL_H / #define GLOBAL_H / #endif`，防止多重包含
3. **模块划分**: 图操作 / 算法实现 / 菜单交互各成独立文件，便于维护和测试
4. **中文支持**: 源文件使用 UTF-8 编码，Windows 控制台若用 GBK，建议使用 `build.bat` 的默认 GCC 编译模式

## 开发记录

- **语言**: C (C99 兼容)
- **依赖**: 仅标准库 (stdio.h, stdlib.h, string.h)
- **内存安全**: 动态分配的内存均有对应的 free 操作
- **输入容错**: `createGraph()` 对非法输入有提示，`locate()` 检查景点是否存在
